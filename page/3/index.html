By Jackey

Exception
定义：运行期间出现的错误
异常是java提供的一种处理程序错误的一种机制
抛出异常：如果程序运行期间出现的异常，可以生成一个异常对象，这个异常对象封装了异常对象的信息并提交给java系统,这个过程称为异常的抛出
捕获异常：当java运行时系统发现异常,会寻找能处理这个异常的代码并把异常交给它处理，这个过程称为异常的捕获。
异常的树形图
1.异常的分类
Throwable : Error,Exception
Error表示JVM虚拟机里面的错误，程序员处理不了
Exception : 其他的异常，RuntimeException
RuntimeException:表示一些经常出现的异常，如 除以0，数组下标越界，等，可以对其不进行处理，当然也可以对其进行处理
其他的异常 ： 表示必须要处理的异常 ， 入 IOException

…可能出现异常的代码块

e.printStackTrance();//打印出错的和出错相关的堆栈信息
e.getMessage();//得到有关异常的信息
//一般来说使用printStackTrance的比较多

}catch(some2Exception e){

e.printStackTrance();
}finally{
//无论是否有异常，都会执行finally里面的代码,
//finally里面通常执行的是文件流的关闭，数据库的回滚等等进行资源的清除工作
}

By Jackey

Published 2017-10-09

java后端面试
校招面试面试官注重的点：
1.基础
2.编程能力

1.String StringBuffer和StringBuilder的区别
(1)在执行速度上，String buffer
其实这是JVM耍的一个把戏 String s = “this is” + “only a” + “fun”;就等于
String s = “this is only a fun”;
（2）线程安全
StringBuffer:线程安全 StringBuilder:线程不安全
虽然StringBuilder的速度比StringBuffer快，但是JVM不能保证StringBuilder在字符串缓冲区操作是线程安全的，但是大部分单线程下操作字符串缓冲区可以使用StringBuilder

总结：对于字符串不频繁的操作使用：String
单线程下操作字符串缓冲区使用;StringBuilder
多线程下操作字符串缓冲区使用：StringBuffer
