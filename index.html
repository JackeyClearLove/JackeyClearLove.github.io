GOD23-设计模式-行为型-观察者模式
观察者模式也被称为：发布-订阅模式，模型-视图模式，源-收听者模式
定义：观察者模式定义了对象间一对多的关系，以便一个对象发生改变的时候，其他的多个对象也要随着更新。

核心
subject ；抽象类，定义了发布者可以增加，删除，提醒观察者更新的方法
concreateSubject ； 继承抽象类，只有设置状态的字段
observer :接口类，只有更新的方法
observerA,observerB :实现了观察者类接口，可以有多个

GOF23-设计模式-行为型-策略模式

在这个案例之中，解决的是问题是：得到各种不同客户的最终价格，算法族是：不同的客户，打折的策略不同

GOF23-decorator

By Jackey

Published 2017-11-25

GOF23-设计模式-结构型模式-装饰者模式
定义：动态的为对象添加新的功能
装饰者模式是一种用于代替继承的技术。无须通过继承增加子类就能扩展对象的新功能，使用对象的关联关系来代替继承关系，更加灵活，同时避免类型体系的快速膨胀。
比如：一面白色的墙，上面粘了挂钩，它就拥有了挂东西的功能，墙上安装了风扇，它就拥有了提供吹风的功能。 墙就相当于初始的对象，而挂钩，风扇就相当于给墙装饰的新功能。

GOF23-composite

By Jackey

Published 2017-11-25

GOF23-设计模式-结构型模式-组合模式
组合模式和java基础里面的组合是不同的
组合是：A类里面定义了B类，那么A类就拥有了B类的属性方法
组合模式：把部分和整体的关系用树形来表示结构表示，从而使客户端可以统一的处理部分对象和整体对象。
组合的核心 ： 抽象构件（component）角色：定义了叶子和容器构件的共同点，
叶子构件角色：无子节点
容器构件角色：有容器特征，可以包含子节点

GOF23-bridage

By Jackey

Published 2017-11-24

GOF23-结构型模式-桥接模式
一般用于商品分类
核心：用于处理多层继承结构，处理多维度变化的场景，将各个维度设计成独立的继承结构，是各个维度可以独立的扩展而不影响其他维度

案例
台式电脑，笔记本电脑，联想台式电脑，神州台式电脑，联想笔记本电脑，神州笔记本电脑
现在就可以建立一个二维，x轴指的是商品分类，y轴指品牌分类，然后用一个桥梁把x,y轴连接起来，并且x,y增加产品可以随意的增加，他们也不会相互影响。

GOF23-设计模式-结构型模式-代理模式
代理模式 ：通过代理，控制对 对象的访问

GOF23-adapter

By Jackey

Published 2017-11-23

GOF23-结构型模式-适配器模式
结构型模式 ：适配器模式 ，代理模式 ， 组合模式，桥接模式，外观模式，享元模式
结构型模式的特点：从程序的结构上实现松耦合，从而扩大整体的类，用来解决更大的问题

定义 ：把一个类的接口转换为客户所需要的接口，适配器把原来不兼容的接口转换为兼容的接口，把原本由于接口不兼容而不能一起工作的哪些类可以一起工作。
模式中的角色
目标接口：客户所期待的接口，目标接口可以是具体或者抽象的类，也可以是接口。
需要适配的类 ： 需要适配的类或是适配者类
适配器 ：通过包装一个需要适配的类，把原接口转换为目标接口

建造者模式经常和工厂模式搭配使用的

比如：现在要组装一个安卓手机，简单的组装安卓手机需要两个步骤：
1.需要组装手机的零件
2.需要组装手机的顺序
总结：相同的组装顺序，不同的零件，组装出来的不一定是同一个商品，零件相同，组装顺序不同，组装出来的东西不一定是同一个商品
建造者模式往往用于构造一些复杂的问题
优点：解耦了组合与零件建造

非静态内部类

内部类也会被编译为独立的 .class文件
格式为 外部类类名$内部类类名.class(demo01$staticclass)

内部类的基本用法
1.静态内部类中不可以调用外部类的普通属性，方法（static的属性）
2.静态内部类和外部类的联系并不大，只是命名空间上的联系
3.在不相关的类中，可以直接定义静态内部类，不需要外部类的干涉
比如：：
class demo02{
publio static void main(String args[]){
//在不想管的类中嗲用内部类
demo01.staticclass ds = new demo01staticclass();
//还可以这样调用
//不过导入包是i这样的 com.*.外部类类名。内部类类名
staticclass ss = new staticclass();

}
}

成员内部类
public class demo03{
private int a = 9;
//成员内部类
public class innerclass{
void test(){
System.out.println(a);
}
}
}
1.可以使用外部类的成员
2.内部类存在是为了辅助外部类(若被其他的类多次使用，可以直接创建为普通的类)
3.成员内部类中不能有静态属性或者方法、 除了final static的属性
在其他不相关的类中调用成员内部类
只能先new一个外部类，在通过外部类调用成员内部类
