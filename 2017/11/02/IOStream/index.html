IOStream理解
IO流包括：字节流 （InputStream,OutputStream）
字符流：（Writer,Reader）
处理流： (BufferReader,BufferWriter)
1.流是单向的，输入流（InputStream）只能用于读（Reader）
输出流（OutputStream）只能用于写(Writer)
2.字节流只能一个字节一个字节的读或者写(不能读写中文)，字符流(一个字符的读写，一个字符等于两个字节，可以读写中文)，他们的返回值都是INT类型的，输出的时候需要强转为字符串类型
3.InputStream,OutputStream,Reader,Writer是所有流的祖宗，所有流都是继承他们（如：FileInputStream,FileOutputStream）
4.BufferReader(Writer)相当于给原来的流（管道）上在包装了一层管道，因为BufferReader里面的readline方法很好用（直接读写一行），并且可以提升读写的效率。
下面看看代码再来理解一下

public class IOTest{

//字节流与字符流并不能完美的解决所有的问题，比如，现在要向内存里面写一个Double类型的数字，当然,把Double转换为字节字符流也是可以实现，但是一个Double或是一个lOng类型的，比如Long占8个字节，那么意味着把long转换为字节要占8个内存，这样一直转很多的long类型的数字那么后果是不堪设想的, 现在就来了解了解DataInputStream和DataOutputStream
//DataInputStream和DataOutputStream是为java原始类型提供存储的流，他存储long(524254254)就是直接把这个数字放到一块内存里面就ok，在不用想字节里面占那么大的内存了，下面看实现的代码
public static void main(String args[]) throws Exception{
ByteArrayOutputStream bos = new ByteArrayOutputStream();
DataOutputStream dos =new DataOutputStream(bos);
try{
dos.writeDouble(Math.random());
dos.writeBoolean(true);
ByteArrayInputStream bis =new ByteArrayInputStream(bos.toByteArray());
DataInputStream dis =new DataInputStream(bis);
System.out.println(dis.available());
System.out.println(dis.readDouble());
System.out.println(dis.readBoolean());

    dos.close();
    dis.close();
}catch(Exception e){
    e.printStackTrace();
}

}
//运行结果
9
0.3342098651358444
true
//先把double和boolean类型写进去，在把它从内存中读出来
//ByteArrayOutputStream外包了一层DataOutputStream，用DataOutputStream的方法写入数据,
//然后ByteArrayInputStream从ByteArrayOutputStream里面读数据，下一步就是向ByteArrayInputStream外包一层DataInputStream，然后利用DataInputStream里面的方法读数据出来

public class testIo {

public static void main(String args[]) throws Exception{
