这个“#{}”，这个是mybatis里面的占位符，相当于mysql里面的 “?” 占位符

这里的id为statement的id，
因为最后会把输入参数，输出类型等的这些数据封装到mapped statement

#{} 占位符
parameterType:输入参数类型
parameterMap:输入参数类型，前期用的这个，现在不用了
· #{id} : 表示要输入的参数，参数的名称为id,如果为简单数据类型，那么括号里面的参数可以随意定
resultType:输出类型，就死查询出来的这条数据是什么类型的，
那么我们查询出来的数据是数据库里面的一条记录，那么对应的就是java里面的一个对象，所以在这里

mybatis开发dao
1.原始的开发方法
2.mapper代理方法开发dao
开发规范
1.ParamType与接口输入参数相同
2.resultType与java接口的返回值相同，如果没有resultType则是void
3.id与java接口的方法名一致
4.nameSpace是java接口的全类名（包+类）

mybatis按照下面的顺序来加载属性
1.先加载properties定义的属性
2.在加载properties元素中resource或者url加载的属性，它会覆盖以加载的同名属性
3.最后加载paramType中的属性

#setting 全局参数配置
这个需要的时候设置，不需要的时候不要乱设置，
这个主要是 ：mybatis运行时可以调整的一些参数，比如，开启一级，二级缓存

#typeAliases 别名（重点）
系统默认的别名

自定义别名
单个别名的定义
typeAliases>
typeAlias type=”com.bl.po.User” alias=”user”/>
/typeAliases>
这里就把User类的别名定义为了user

批量别名的定义
typeAliases>
package=”com.bl.po”/>
/typeAliases>
系统自动定义别名
系统会把package包里面的类的别名定义为类名，首字母大写小写都可以

#typeHandler
类型处理器，通过类型处理器，可以完成java导 jdbc类型的处理，一般系统默认支持的就狗我们使用了，不需要来自定义。

#映射器mapper
加载单个的映射文件

mapper resource="conf/sqlmap/user.xml"/>

通过接口加载mapper
mapper class=”com.bl.mapper.User”/>
需要遵循的规范
1.mapper.xml和接口的文件名称一致
2.mapper.xml和对应的接口在同一目录下

mapper url=”file:/c:/….”/>
url里面放的就是xml文件的物理路径

批量的加载mapper.xml文件
《package name=”包名”/>
这样就会批量的来加载这个包下的mapper文件
遵循的规范
1.mapper.xml和接口的文件名称一致
2.mapper.xml和对应的接口在同一目录下
这个规范必须要遵循的

public class userMapperImpl implements userMapper{
private static SqlSessionFactory sessionFactory ;
static{
try{
String path = “conf/sqlMapConfig.xml”;
InputStream inputStream = Resources.getResourceAsStream(path);
sessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
}catch(Exception w){
w.printStackTrace();
}
}

@Override
public List<customIfo> findUserList(CustomVo customVo) throws Exception {
    SqlSession session = sessionFactory.openSession();
    userMapper mapper = session.getMapper(userMapper.class);

    CustomVo customVo1 = new  CustomVo();
    customIfo cus1 = new customIfo();
    cus1.setName("blld");
    cus1.setEmail("243808244@qq.com");
    customVo1.setCus(cus1);
    List<customIfo> userList = mapper.findUserList(customVo1);

    return userList;
}
