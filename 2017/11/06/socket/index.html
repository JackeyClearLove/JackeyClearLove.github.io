socket :插座
在各个语言中，网络编程都被称为socket编程
网络分层 ： 应用层–传输层 –网络层–数据链路层
tcp(translation control protocol) 在不可靠的网络中提供可靠的网络协议(协议是，相当于中国说汉语，那么汉语就是中国人的协议，那么英国人交谈的协议就是英语)。
可靠 ：比如打电话 ， A与B建立连接 A：喂喂喂。必须要听到B的回话，否则A就一直喂喂喂，直到B回话，才对B说想要说的内容， 可靠就是建立连接之后，我给你发送数据包，你必须回应我你收到了，我才给你发送下一个。
UDP（user data protocol） 不可靠的网络协议
不可靠 ： 就是我只负责发送包，不管你是否收到，反正我只负责发送。
场景： 比如我们玩英雄联盟，我的伊泽瑞尔发出一个Q，但是因为网络延迟的原因，我的Q技能本来是0.5秒就应该命中敌人，但是却用了3秒，敌人本来就是残血，但是因为网络延迟，没有看到你的技能，还在找位置对你输出，3秒后你的技能击中敌人，你莫名其妙的杀死了对方。这就是用UDP传输的，在UDP传输的过程中，偶尔出现丢包，也是没有多大问题的。
安全性 tcp > udp 传输速度 ： udp > tcp

端口 ：在一台计算机中，最多可以有65536个端口，在设置端口的时候，最好设置在1024以上的端口，因为1024以下的端口可能被系统征用。

TCPServer.java

public class TCPServer {

public static void main(String[] args) throws Exception{
    ServerSocket ss =new ServerSocket(6666);
    Socket s = ss.accept();
    System.out.println("client connect....");
    InputStream is = s.getInputStream();
    DataInputStream dis =new DataInputStream(is);
    String res = dis.readUTF();
    System.out.println(res);

}

}

TCPClient.java
public class TCPClient {

public static void main(String[] args)throws Exception {
    Socket ss =new Socket("127.0.0.1",6666);
    OutputStream os =ss.getOutputStream();
    DataOutputStream dos = new DataOutputStream(os);
    dos.writeUTF("hello,socket");

}

}

socket必须有客户端与服务端
服务端里面需要有一个端口（自己定义），等待指定的客户端去连接，如果没有人连接，他就一直等着客户端连接，连接之后，余姚ServerSocket里面的accept方法去接受过来的链接
对他们的对话需要用流去完成，上面的DataInputStream是在getInputStream外面包裹的一层流，这里是想用到DataInputStream的readUTF方法采用的，你也可以用其他的流去包裹getInputStream();
