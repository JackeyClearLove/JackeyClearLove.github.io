GOF23-设计模式-结构型模式-组合模式
组合模式和java基础里面的组合是不同的
组合是：A类里面定义了B类，那么A类就拥有了B类的属性方法
组合模式：把部分和整体的关系用树形来表示结构表示，从而使客户端可以统一的处理部分对象和整体对象。
组合的核心 ： 抽象构件（component）角色：定义了叶子和容器构件的共同点，
叶子构件角色：无子节点
容器构件角色：有容器特征，可以包含子节点

//这个就是组合模式的一个大概实现思路
interface component{
void oprate();
}

interface composite extends component{
void oprate();
void add();
void remove();
component get(index);
}

interface leaf extends component{
void oprate();
}

//下面以杀毒的案列来举例组合模式
抽象接口 对应的上面的component
public interface abstractFile {
//杀毒
void killViues();
}
//叶子构件
public class leafFile implements abstractFile{
//这个使叶子节点，叶子节点下面无节点
private String name;
public leafFile(String name){
this.name =name;
}
@Override
public void killViues() {
System.out.println(“文件”+ name +”正在杀毒….”);

}

}
//容器构件
public class compositeFolder implements abstractFile{
List list = new ArrayList();
private String name;
public compositeFolder(String name){
this.name = name;
}
@Override
public void killViues() {
System.out.println(“文件” + name +”正在杀毒。。。。”);
for(abstractFile File:list){
File.killViues();
}

}

public void add(abstractFile file){
list.add(file);
}
public void remove(abstractFile file){
list.remove(file);
}

public abstractFile get(int index){
return list.get(index);
}

}

//测试
public class Client {

public static void main(String[] args) {
    leafFile f3,f4;
    compositeFolder c = new compositeFolder("我的收藏");

    f3 = new leafFile("视频软件");
    f4 = new leafFile("看图软件");
    c.add(f3);
    c.add(f4);
    c.killViues();
}

}

//结果 ：
文件我的收藏正在杀毒。。。。
文件视频软件正在杀毒….
文件看图软件正在杀毒….
容器构建里面存在一个天然递归（自己调用自己）

可以看到，在客户端，只需要调用killViues,就可以完成对容器构建下的叶子的一系列处理

容器：有节点，由叶子构成
叶子：无节点，是容器的组成部分

1.操作系统的资源管理器
2.GUI中的容器层次图
3.XML文件解析
4.OA系统中，组织结构的处理哦
5.Junit单元测试框架
