GOF23-decorator

By Jackey

发表于 2017-11-25

文章目录

GOF23-设计模式-结构型模式-装饰者模式
定义：动态的为对象添加新的功能
装饰者模式是一种用于代替继承的技术。无须通过继承增加子类就能扩展对象的新功能，使用对象的关联关系来代替继承关系，更加灵活，同时避免类型体系的快速膨胀。
比如：一面白色的墙，上面粘了挂钩，它就拥有了挂东西的功能，墙上安装了风扇，它就拥有了提供吹风的功能。 墙就相当于初始的对象，而挂钩，风扇就相当于给墙装饰的新功能。

实现细节
component(抽象对象)
concreateComponent(真实对象)
decorate(抽象的装饰)（持有对真实对象的引用）
concreateDecorate(真实的给对象装饰的新的功能)
eg :
component ; ICar
concreateComponent: car implement ICar
decorate :decorateCar 1.对真实对象Car的引用 2.装饰功能的抽象
concreateDecorate ：fly implements decorateCar,warter implements decorateCar

下面以实列代码帮助理解
ICar接口
public interface ICar {
void move();
}
//真实对象
public class Car implements ICar{

@Override
public void move() {
    System.out.println("陆地上跑");

}

}
//装饰对象
public class decorateCar implements ICar{
private ICar car;
public decorateCar(ICar car){
this.car = car;
}
@Override
public void move() {
car.move();

}

}

//具体功能
//飞的功能
public class flyCar extends decorateCar{

public flyCar(ICar car) {
    super(car);
}
public void fly(){
    System.out.println("天上飞");
}
@Override
public void move() {
    super.move();
    fly();
}

}
//水里游的功能
public class warterCar extends decorateCar{

public warterCar(ICar car) {
    super(car);
    // TODO Auto-generated constructor stub
}
public void warter(){
    System.out.println("水里游");
}
@Override
public void move() {
    super.move();
    warter();
}

}

//测试类
public class Client {

public static void main(String[] args) {
    System.out.println("原始未装饰的对象");
    Car c=new Car();
    c.move();

    System.out.println("--装饰了飞的功能--");

    flyCar f = new flyCar(c);
    f.move();

    System.out.println("--装饰了飞和水的功能--");
    warterCar w =new warterCar(new flyCar(c));
    w.move();
}

}

//运行结果

原始未装饰的对象
陆地上跑
–装饰了飞的功能–
陆地上跑
天上飞
–装饰了飞和水的功能–
陆地上跑
天上飞
水里游
这样就实现了给car对象装饰多个功能，并且可以随时增加功能扩展car的功能，而不用修改原先的代码，遵循了设计模式的开闭原则

开发中使用的场景
IO流
SWing包中图像构件的功能
servlet中的request(httpRequstServlet)
struts2中对request,response,session的处理
