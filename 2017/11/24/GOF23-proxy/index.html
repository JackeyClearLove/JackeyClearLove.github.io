GOF23-设计模式-结构型模式-代理模式
代理模式 ：通过代理，控制对 对象的访问
初学者对这个定义可能会产生模糊或者混淆，现在我把我的理解与大家分享

案例：场景 ：买鞋 人物对象 ：鞋生产厂商，鞋厂销售，买鞋的人
鞋厂销售就相当于 鞋厂生产厂商的代理 ，鞋厂只需要关注生产鞋子，卖鞋子的问题就可以交给代理，代理把从鞋厂的鞋子拿出去给顾客销售， 这样，鞋厂 就通过 代理（鞋厂销售），完成了对 顾客（访问对象）的控制
这只是一个小列子，可能会有更加简介清晰的列子，学习的人，可以借助这个案列去理解代理模式，而不要依赖这个列子或者死记硬背，每个人都有自己的学习方法，自己总结并理解的东西，才是自己的。

代理： 静态代理，动态代理
动态代理的典型案列（spring aop）

代理中的可信角色
1.真实角色 2.代理角色 3.抽象角色 4.客户类
角色之间的联系 ：客户类通过抽象角色（代理角色实现抽象角色）实现对真是角色的访问

以上面的列子来理解这个核心的角色 ： 顾客 通过 代理 完成对 鞋厂的访问
将统一的流程放到代理中去实现

静态代理
以歌星唱歌的案列来演示静态代理模式
比如：个性唱歌需要这样一套流程
1.签约 2.面谈 3.唱歌 4.收款
这一套流程就是我们抽象方法中需要定义的
下面以代码的形式帮助理解
//抽象接口
public interface abstartInterface {
//现在模仿歌星唱歌的一套流程模范
//签约
void sign();
//面谈
void faceToface();
//唱歌
void singSong();
//收款
void payMoney();
}

// 真实歌星
public class realStart implements abstartInterface{

@Override
public void sign() {
    System.out.println("代理处理");

}


@Override
public void faceToface() {
    System.out.println("代理处理");

}

@Override
public void singSong() {
    System.out.println("周杰伦唱歌");

}

@Override
public void payMoney() {
    System.out.println("代理处理");

}

//歌星代理
public class startProxy implements abstartInterface{
realStart rs;
public startProxy(realStart rs){
this.rs = rs;
}

@Override
public void sign() {
    System.out.println("签约");

}

@Override
public void faceToface() {
    System.out.println("面谈");

}

@Override
public void singSong() {
rs.singSong();

}

@Override
public void payMoney() {
    System.out.println("收款");

}

//客户类
public class Client {

public static void main(String[] args) {
    realStart real = new realStart();
    startProxy proxy = new startProxy(real);
    proxy.sign();
    proxy.faceToface();
    proxy.singSong();
    proxy.payMoney();
}

}

//结果
签约
面谈
周杰伦唱歌
收款
ok,歌星实际就唱了歌曲，其他的都交给代理去完成

动态代理
动态代理相比于静态代理的优点 ： 动态代理 把 静态代理中的 抽象类（接口类）统一交给了处理器中的一个集中的方法中去处理，这样我们可以更加灵活的处理众多的方法。

Proxy
java.lang.reflect.Proxy
作用：动态生成代理类对象
InvocationHandler
Java.lang.reflect.InvocationHandler
可以通过invoke()方法实现对真实角色的访问
每通过Proxy代理类创建一个代理对象都要有一个对应的处理器
下面用代码来帮助理解
//被代理的接口
public interface Start {
void singSong();
}

//实现被代理的接口
public class realStart implements Start{

@Override
public void singSong() {

    System.out.println("周杰伦唱歌");

}

}

//处理器
public class StartHandler implements InvocationHandler{
private realStart real;
public StartHandler(realStart real){
this.real = real;
}

@Override
public Object invoke(Object proxy, Method method, Object[] args)
        throws Throwable {
    System.out.println("方法执行前");
    method.invoke(real, args);
    System.out.println("方法执行后");
    return null;
}

}
//测试
public class test {

public static void main(String[] args) {
    realStart real = new realStart();
    StartHandler startHandler = new StartHandler(real);
    Start newProxyInstance = (Start)Proxy.newProxyInstance(ClassLoader.getSystemClassLoader(), new Class[]{Start.class}, startHandler);
    newProxyInstance.singSong();
}

}
//结果

方法执行前
周杰伦唱歌
方法执行后
这样在 方法执行前后都可以做处理，这些就是静态代理中，代理帮真是角色做的事情被代替后

基本上框架都用的代理模式
Proxy中的newProxyInstance()方法
原理：1.把真实的角色类通过javassist(CGLIB)（这个就是类加载器）生成字节码文件到本地
2.通过反射得到生成字节码文件中的方法、
3.在方法的前后做处理
这样就可以实现方法前后做处理
当然，JDK里面也是这个原理，但是JDK里面多了很多的细节处理而已
