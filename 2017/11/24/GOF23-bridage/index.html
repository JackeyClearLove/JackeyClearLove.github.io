GOF23-结构型模式-桥接模式
一般用于商品分类
核心：用于处理多层继承结构，处理多维度变化的场景，将各个维度设计成独立的继承结构，是各个维度可以独立的扩展而不影响其他维度

案例
台式电脑，笔记本电脑，联想台式电脑，神州台式电脑，联想笔记本电脑，神州笔记本电脑
现在就可以建立一个二维，x轴指的是商品分类，y轴指品牌分类，然后用一个桥梁把x,y轴连接起来，并且x,y增加产品可以随意的增加，他们也不会相互影响。

下面用代码实现
//品牌
public interface brand {
void sale();
}
class lenove implements brand{

@Override
public void sale() {
    System.out.print("卖联想");

}

}

class shenzhou implements brand{

@Override
public void sale() {
    System.out.print("卖神州");


}

}

//电脑
public class computer{
protected brand b;
public computer(brand b){
this.b = b;
}

public void sale(){
    b.sale();
}

}
class destop extends computer{

public destop(brand b) {
    super(b);
    System.out.println("买台式");
}

}

class latop extends computer{

public latop(brand b) {
    super(b);
    System.out.println("买笔记本");
}

}

//测试类
public class Client {

public static void main(String[] args) {
 computer co= new destop(new lenove());
 co.sale();
}

}
这里就是通过接口brand来把品牌和样式机两个维度连接起来的，这样无论你是添加品牌还是添加电脑类型都对彼此不会有影响，这是组合的方式。。。。
