面向对象的再次回顾与感悟
1.类：类是对象的一个模板,对象只是类中的一个实列，比如：狗是一个类,黑狗是狗类的一个被实列化的对象
2.继承，聚合，组合： 判断什么时候用什么 。 当xx是一种xx的时候,用继承(猫是一类动物)
当xx是xx的一部分(足球是球类的一部分)用聚合 当xx是xx密不可分的一部分用组合(头是人体密不可分的一部分)
3.八中基本数据类型:double float int short long byte char blooean
4.引用类型：出了上面的八种,全部都是引用类型
5.内存分析：String s, s=new String(); s是放入stack里面的，为null,newString()是放在heap里面的,s是对new String()的一个引用,对象的成员变量都是放在heap里面，例如：
public class Person{
int i;
int j;
public static void main(String args[]){
Person p1 =new Person();
Person p2 = new Person()
}
}

在调用的时候, p1.i , p1.j p1 p2是 放在stack里面的,p1.i p1.j是放在heap里面的,p1,p2两个对象之间的成员变量是不同的,p1,p2共享一个类的同一个方法
6.构造函数：构造函数分为无参构造函数与有参构造函数,
public class Person{
int i;
int j;
Person(){

}
person(int _i,int _j){
i = _i;
j = _j;
}
}
构造函数的意思是：构造一个自己的对象
当我们Person p =new Person(i,j);的时候，相当于调用的这个类的构造函数，无参构造函数如果不写，系统会自动为我们创建。
有参的构造函数的传值情况是_i传值给i,_j传值给j,
7.方法重载：表示方法名相同，但是参数不相同的同一类里面的多个方法。注意：返回值不同，参数名相同的，不能构成方法重载。 构造方法也可以构成方法重载
public class Person{
String name;
int age;
//这两个是构造方法的重载
person(int age,String name){
this.age = age;
this.name = name;
}

person(String name){
this.name =name ;
this.age = 20;
}
//下面的是方法重载
void max (int i,int j){
System.out.println(a>b?a:b);
}

void max (double i,double j){
System.out.println(a>b?a:b);
}
}
8.this关键字 this关键字表示的是对象本身，是对对象本身的一个引用(程序执行内存的分配是解决问题的根源所在,搞懂了程序执行内存是怎么分配的,一切就都明了了)
9.static关键字：前面带static的方法或成员变量表示为静态方法或静态变量，
Public class jt{
static String ss;
String ss1;
}
这个jt j =new jt();当这个对象new了一次之后 ,ss1在heap里面有一个并且是对象new 一次生成一个ss1，ss在data seg里面有一个并且不管对象new多少次,都只有一个ss,static是属于类级别的,静态方法里面不能调用非静态的方法
10.J2SDK中主要的包介绍：

12.equals 与 == 的区别
public class Cat{
int height;
int weight;
public Cat(int height,int weight){

}
}
public class test{
public static void main(String[] args){
Cat c1 =new Cat(1,2);
Cat c2 = new Cat(1,2);
System.out.println( c1 == c2);
}
}
返回值 ： false
明明就是new的同一个Cat,参数也是传的一样,那么为什么返回值还是fasle呢？ ok ,接下来就说说 == 与equals的不同
== ：c1 和c2 在内存中new出来的是处于不同的地址， == 号比较的是地址是否相同，所以返回是false
Ok 那么我们把 System.out.println( c1 == c2);改为System.out.println( c1.equsls(c2) );
在run as，但是，输出的依然是false,那这又是为什么呢，
equals : 比较的是内容是否相同,但是同一个对象不同的引用，返回的也是false
也可以这么说，虽然是同一个类，但是当c1 == c2为false的时候,那么c1.equals(c2)也永远为false.
String等等的一些类中都是重写了equals方法

13.向上转型(upcasting):把子类转换为父类的类型
向下转型(downcasting)：把父类转换为子类的类型
eg:
public class Animal{
Sting name;
}
public class Cat{
String height;
public Cat(int height){
this.height = height;
}
}

public class Test{

public static void main(String args[]){
Animal a1 =new Animal();
Cat c1 = new Cat();
Animal a2 = new Cat(1);
//System.out.println(a2.height);//这个是向上转型，那么转型之后a2就只能看见Animal里面的成员变量，而不能访问Cat里面的成员变量
Cat a3 = (Cat)a2; //这个是向下转型,a3可以访问Animal里面的方法和Cat里面的放大
}
}

15.多态：指不同类的对象可以对同一消息做出响应，也就是同一个消息发送的对象不同而采用不同的行为方式(发送消息就是调用函数或者说调用方法)
多态的实现机制：动态绑定
多态的条件：1.继承 2.重写 3.父类引用指向子类对象
16.final: final的变量不能被改变 final 的方法不能被重写 final的类不能被继承
17.接口（interface） 1.接口里面的方法都是static final写不写都是这样的 2.接口里面的修饰符默认是public的 3.接口里面的方法都是抽象的，声明不声明都是抽象的 4.接口可以实现接口，在实现的接口里面继续定义新的属性和方法
