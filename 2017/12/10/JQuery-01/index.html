#JQuery的编程方式
JQuery是采用链式编程的方式，什么是链式编程，下面用java代码来简单的了解下
public class test {

public static test m1(){
    return null;
}

public static test m2(){
    return null;
}

public static void main(String args[]){
    test.m1().m2();
}

}

//这个就是一个简单的链式编程，test类里面的方法返回的都是test类型，所以调用的时候可以一直用 “ . “的方式一直调用，形成了一条编程链。

    ##层次选择器
    一般层次选择器和表单选择器都是查看着使用，基本选择器是最常用的。
    意义：通过DOM对象中的层次关系来获取元素，通常包括 后代，父子，相邻，兄弟关系。通过其中某类的关系，可以方便的定位元素。

    ancestor descendant 根据祖先元素匹配所有的后代元素。 集合
    parent > child 根据父元素匹配所有的子元素。 集合
    prev + next 匹配所有紧接在prev元素后面的相邻元素 集合
    prev + siblings 匹配prev元素之后的所有兄弟元素。 集合

    div id=”div1”>left div>
    div id=”div2”>
    span id=”span1”>span1 span/>
    span id=”span2”>span2 span/>
    div/>
    span id=”span3”>span3 span/>
    div id=”div3”>left1 div>
    div id=”div4”>left2 div>
    div id=”div5”>left3 div>

    ancestor descendant：这个就相当于包含关系，
    比如 $(‘#div2 span’) :这个选择的就是div2下所span

parent > child ：匹配子元素
eg : $(‘#div2 > span’) : 匹配div2下的子元素（只匹配一层）

prev + next： 这个就是选择下一个元素，它可以分为两个
第一个： $(‘#div2 span’)。next ,这个是选择div2下的span（这个span必须是兄弟元素或者说是相邻元素）
第二个 ： $(‘#div2 + span’).next : 这个相对上面的就严格了，它规定span元素必须要在div2的下面，否则不会被选择。

prev + siblings：这个就是匹配除了自己的所有的兄弟元素
eg: $(‘div2’).sibling(‘div,span’)： 他会匹配所有的(div 和 span)兄弟元素

$(div2 ~ div) :匹配div2后面的所有div元素

一般层次选择器用的还是比较少的。

#过滤选择器

##简单选择器
过滤选择器根据某类过滤规则进行元素匹配，书写都是一（：）开始，简单过滤选择器是过滤选择器中最常用的一种。
:firt 选择第一个元素
last 选择最后一个元素
not(selector) 匹配除了给定选择器之外的元素 集合
odd :even 索引 从0开始，索引值为奇（偶数）数的元素 集合
:eq（index） 索引从0开始，获取指定索引值的元素
:gt(index) 索引从零开始，获取大于给定索引值的元素
:lt(index) 索引从零开始，获取小于给定索引值的元素
:header 获取所有标题元素，如h1,h2等
:animated 获取正在执行动画的元素
注意： “ : “ 与关键字之间不能有空格

##内容过滤选择器
:contains(text) 获取包含给定文本的元素
:empry 获取空文本的元素
:has(selector) 获取含有选择器匹配元素的元素
:parent 获取含有子元素（文本）的元素

##可见性选择器
:hideen 获取所有不可见的元素或者type为hidden的元素
:visiable 索取所有可见的元素

##属性过滤选择器
属性过滤选择器在过滤选择器里面是最常用的
[attribute] 获取包含给定属性的元素
[attribute = value] 获取给与给定属性值相等的元素
[attribute !=value] 获取与给定属性值不等的元素
[attribute^=value] 获取与给定属性值开始的元素
[attribute$=value] 获取与给定属性值结尾的元素
[attribute*=value] 获取与给定属性值是以包含value值的元素
[selector1][selector2][selector3] 获取满足多个条件的复合属性的元素

##子元素过滤选择器
:nth-child(eq|even|odd|index) 索引从1开始，获取每个父元素下特定位置的元素
:first-child 获取父元素下的第一个孩子
:last-child 获取父元素下的最后一个孩子
:only-child 获取每个父元素下只有一个子元素的元素

##替换节点
JQuery中元素的替换主要有两种方式：
replaceWith()和replaceALL()
两者唯一的区别在于replaceWith()是用括号中的内容替换指定的元素，
replaceAll是指定的元素被括号中的内容替换
-targerElement.replaceWith();
$().replaceAll(targetElement);
$(“#p1”).replaceWith(“元素标签的内容”);
$(“元素标签的内容”).replaceAll(“#p1”);
注意： 一旦完成替换，则原来被替换的元素全部消失。

##包裹节点
JQuery中通过包裹某个节点，是其增加响应的功能.主要的方法有

wrapInner(html) //包裹此节点的内部元素（包括文本内容）

##遍历元素
在传统的js中，要实现遍历元素需要获得总长度然后for循环去遍历，或者用for(var i in elements){}去遍历。而在JQuery中则不用那么麻烦，直接用each()方法即可。
-each(callback) //callback回调函数
注意： callback是一个函数，该函数内部有一个形参index,此形参为遍历元素的序号(index 从0开始)。如果需要访问属性可以通过index,配合this关键字使用。
$(“input”).each(function(index){
this//表示选择器选中的元素
this.val;
})

##删除元素
DOM操作时，删除多余或者指定的页面元素是非常必要的。JQuery提供了2中可以删除元素的方法：remove()和empty(),严格来说，empty()方法是清空元素节点和后代节点
removw(expr)
注意： 这里的 expr为可选项，表示过滤删除的意思，可匹配选择器，如果不加参数则表示删除当前元素节点。所以JQuery的remove()方法还是十分强大灵活的。
这个删除很强大。里面添加的过滤条件可以让我们随心所欲的做很多东西。
