JUnit单元测试
什么是单元测试？
比如：我写了个类，我要确定这个类是否正确，那么就需要测试一下，
比如
public class t{

public int add(int a,int b){
return a+b;
}

//测试就是你调用这个类的add方法，然后给a,b赋值，如果测试出来的与我们想要的结果相同，那么测试成功，证明我们写的程序是正确的。
}

现在写一个简单的方法测试
还用上面的列子
public class T {

public int add(int a,int b){
    return a+b;
}

}

//测试
import static org.junit.Assert.*;

public class TTest {

@Test
public void testAdd() {
    fail("Not yet implemented");
    int z = new T().add(3, 5);
    assertEquals(8, z);
}

}

//import static org.junit.Assert.*; 这个不是一个包，是静态引入的方法，
assert断言，就是判断的意思。

JUnit4里面出来了新的断言
assertThat
用了这个断言，那么一切的测试都是可以搞定的
这个属于hamcrest静态方法里面的
具体各个都有什么断言，可以参考具体的API文档
assertThat(n,allOf(greatThan(1),lessThan(1)));//n必须满足allOf里面的所有条件才算通过。
assertThat(n,anyOf(greatThan(1),lessThan(1)))//满足其中的任何一个条件。
assertThat(n,anything());//n是什么值都可以测试通过。
assertThat(n,is(1));//n是不是1，是1就通过
assertThat(n,not(1));n是不是 不是1，n不是1，测试通过

assertThat(n,containString(“fff”));//n是不是包含fff这个字符串
assertThat(n,endWith(“bb”));//n是不是以bb结尾

assertThat里面定义中代码的作用和英文的意思很切近，如果英文稍微可以，多看看，多测试测试还是没有太大的问题的。
这些都是一些基础，实用性的比如测试struts,Spring等一些项目中遇到的问题，但是要去测试这些复杂的问题，基础还是很重要。

Failure:表示测试错误。
error ；表示程序本身错误。

#JUnit Annotation
@Test(expected=java.lang.Excetion,timeout=100)//表示这个方法是一个测试方法,expected表示期望干什么，这里写的期望抛异常，那么如果有异常，测试依然可以通过。
timeout表示你期望这个方法在多少毫秒内执行完成，这个可以用于测试效率。

@Ignore 表示本次测试不测试这个方法。（比如在项目中，某个模块还没有写完，条件还不能满足，那么我们可以使用这个注解暂时忽略不测试这个方法）

@before 所有的方法执行值钱都要执行这个方法
@After 所有的方法执行之后都要执行这个方法
public class T{
@before
public void before(){
System.out.println(“before”);
}
@Test
public void tt(){
…..
}

@Test
public void bb(){
…..
}
@After
public void before(){
System.out.println(“after”);
}
}

//结果
before
after
before
after

//应用，比如我执行测试方法里面的代码需要先打开某个文件，执行完后需要关闭某个文件

@beforeClass //所有测试开始之前运行
@afterClass //所有测试执行结束之后运行

这两个注解的方法必须是static的，否则不能运行。
应用，：当我们需要在测试之前搭建很耗时的环境，加载配置文件的时候。
