nginx反向代理

正向代理：eg:在一个教室里面有老师和学生,学生的电脑是没有网的，但是老师的电脑有网，那么学生就可以通过老师的电脑去上网，这个过程中，学生发出请求，老师的电脑把学生的请求发出去，服务器得到结果，把结果返回给老师，老师再把结果返回给学生。nginx做的就是一个请求转发的作用。

反向代理：
server{
upstream baidu{
server 192.168.176.132:8080;
}
listen 80
hostname www.baidu.com
location /{
proxy_pass baidu
index index.html index.html
}

}

server{
upstream jd{
server 192.168.176.132:8081;
}
listen 80
hostname www.jd.com
location /{
proxy_pass jd
index index.html index.html
}

}
用户请求–》nginx区分访问的是www.baidu.com还是www.jd.com
如果访问的是www.baidu.com
那么proxy_pass代理会找到upstream baidu然后访问192.168.176.132:8080
8080端口的tomcat把返回的页面给nginx,nginx再把页面(结果)给用户

负载均衡：给upstream里面添加tomcat服务器 ./nginx -s reload
用户每次访问 就会轮询这个域名里面的每个服务器
新服务器和老服务器的问题
新服务器的性能好 老服务器的性能较差
那么我们可以在nginx的upstream里面设置权重
权重越高 被访问到的概率越大

高可用
keepalived+nginx
keepalived通过询问nginx的心跳来判断nginx是否还活着
若果宕机，keepalived就会把宕机的ip地址绑定导自己身上
然后充当主机的角色
如果主机修好之后，主机就会把备用机的ip拿过来，继续工作

nginx可以做的事情
访问静态资源
反向代理
负载均衡（轮询，权重）
高可用
