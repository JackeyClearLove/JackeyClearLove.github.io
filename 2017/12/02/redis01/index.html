redis使用

redis的安装以及基本的命令操作

在linux上安装 利用sftp
解压
tar -zxf redis-3.0.0
redis是用c写的
编译之前要确定有gcc环境
如果没有
yum install gcc-c++

安装执行这个命令 会更新你的gcc环境
没有安装：给你安装 gcc环境
\
编译
make
安装到指定的目录
make install PREFIX=/usr/lcoal/redis
拷贝
cp redis.conf /usr/local/redis/bin/
编辑配置文件
vim redis.conf
把deamosize no 的no改为yes(应该是静默的意思)
启动redis服务
./redis-server redis.conf
查看服务是否启动
ps -aux|grep redis
出现两行 则表示服务已经启动

关闭服务
kill (ps里面的进程)eg 5112
或者 redis-cli shutdown也可以关闭
同样也可以利用
ps -aux|grep redis查看服务是否关闭

客户端连接服务
./redis 回车
出现127.0.0.1：6379
我们这个改为自己虚拟机的ip
./redis-cli -h 192.168.25.128 -p 6379
这样就会哦出现
192.168.25.128:6379>
然后你就可以开始玩reids了

redis的基本命令
redis 的5种基本数据类型
strings(字符串) hashs(哈希) lists(列表) sets(无序集合) sorted sets(有序集合
)
String
set str1 abc
get str1
返回结果abc
keys 或 key ““(查看所有的key)
删除
del str1
若 不存在返回(integer) 0
删除成功返回 (integer) 1
incr key1 原子+1
decr key1 原子-1
incrby key1 integer 原子+integer
decrby key1 integer 原子-integer

hash
hset hash1 field 1
hash1 表示的是redis的key field1表示hash的key 1表示hash的value
hget hash1 field1
返回值为 1
hgetall hash1
返回hash1下面的所有的hash的key与value
hdel hash1 field1
删除hash的key为field的key

list
lpush list1 1 2 3 4 5 6
表示从左面开始添加
lrange list1 0 -1
查看list1的所有值
6 5 4 3 2 1
rpush list1 a b c d e
从右边开始添加
lrange list1
a b c d e
lrange list 3 9
表示查询list1里面3-9数据
lpop list1
取出并移除第一个元素
rpop list1
去除并移除最后一个数据、
lrem list1 count value
删除count个value
lrem list1 3 1(表示删除list1里面的3个为1的value)
ltrim list1 start end
清空索引在start和end之外的元素,索引从0开始
eg 1 2 3 4 5 6 7 8
ltrim list1 2 4
返回ok
lrange list1 0 -1返回的结果为3 4 5

sets
添加集合
sadd seta a b c d e f
sadd setb c d e g h k
求差集合
sdiff seta setb a-b
结果 a b f
sdiff setb seta
结果 g h k
交集
sinter seta setb
结果：c d e
并集
sunion seta setb
结果 a b c d e f g h k

sorts sets(有序集合)
zadd
zrevrange
zscore key number

expire key secod(设置key的过期时间)
ttl key (查看key的有效期)
正数：过期时间的倒计时
-1：永久有效
-2：无效的key
presist key 持久化

redis的持久化
Rdb(快照形式，默认开启) 定期的把内存中的数据保存到磁盘中
aof形式 append only file 把所有对redis执行的命令（crud操作命令）保存导文件中、数据库恢复时，把所有的命令执行一遍

redis集群

在/usr/local/redis-cluster下copy 6个redis下面的bin
bin里面必须是空的 也就是不能有.rdb(快照形式) .aof(append only file)持久化的方式，若果有这个后缀的文件必须删除掉

  1. 那么现在在 redis-cluster下面就有redis01 ….redis06 6个文件
    修改redis01-redis06里面的redis.conf里面的port分别改为7001-7006
    把注释的cluster redis-cluster enable(在文件中被注释掉了) 放行
    这样文件就修改好了
    2.在redis-cluster下面创建一个start-allserver.sh
    vim start-allserver.sh
    cd redis01
    ./redis-server.sh redis.conf
    cd ..
    cd redis02
    ./redis-server.sh redis.conf
    cd ..
    cd redis03
    ./redis-server.sh redis.conf
    cd ..
    cd redis04
    ./redis-server.sh redis.conf
    cd ..
    cd redis05
    ./redis-server.sh redis.conf
    cd ..
    cd redis06
    ./redis-server.sh redis.conf
    cd ..

    esc :wq保存

3../start-allserver.sh 启动这6个redis服务
ps aux|grep redis 查看进程这些服务是否已经被启动

4.在/root/redis-3.0.0/src目录下把redis-trib.rb
cp 到 /usr/lcoal/redis-cluster这个目录下

cp redis-trib.rb /usr/lcoal/redis-cluster -r

5.运行这个文件需要安装ruby环境
yum install ruby
yum install rubygems
6.在windows中把redis-3.0.0.gem(这是一个第三方的jar文件)这个文件通过sftp传输到/root下
执行命令
gem install redis-3.0.0 回车 选择yes

  1. ./redis-trib.rb create –replicas 1 192.168.25.128:7001 192.168.25.128:7002 192.168.25.128:7003 192.168.25.128:7004
    192.168.25.128:7005 192.168.25.128:7006 回车
    系统就会为你搭建集群
    上面这个命令的意思是
    –replicas 1 表示每个主机有一个备份机
    那么就有三个node 每个node都有1个备份集
    所以搭建集群的时候需要偶数个redis
    系统会为每个主机分配槽（slot 范围：0-16383）
    slot的分配是根据 crc16算法为key计算出一个值
    然后16384与这个值的余数就是各个槽的分配结果

奇怪的是 分配完 问你是否同意对这么槽的分配
妈的 不同意有毛办法啊 我又改不了
只能yes
