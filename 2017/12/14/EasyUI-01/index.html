##简单的消息框
$.messager.alert(‘Warning’,’The warning message’);

$.messager.confirm(‘Confirm’,’Are you sure you want to delete record?’,function(r){
if (r){
alert(‘确定’);
}else{
alert(“取消”);
}
});
//这个是easyui提供给我们简单的消息框，这个和js中的alert是不同的，通过测试，alert()会阻止后面的消息执行，也就说如果alert后，如果不点击确定按钮，那么后面的程序是一直处于等待状态的，但是esayui的messager.alert不会阻止程序的执行。
当然还有其他的消息框，比如输入，右下角弹出提示框（过段时间自动消失），等

从api中就可以看到，panel是个顶层容器，window依赖yupanel,dialog依赖于window,这里的依赖就相当于java中的继承，也就是子类拥有父类的所有方法。那么就可以想象，dialog其实是可以使用panel和window的所有的属性的（那么dialog就很强大），但是使用什么 还是要根据具体的场景去使用相应的组建，具体怎么使用，可以参考api文档，里面就写的非常清楚。

组建的写法：
第一：纯html
第二：js或者js和html结合，都是可以的
引入只需要 class=”easy-panel[window或dialog]”

动态加载组建：
easyloader.load(‘messager’,fn(){})
messager:指定的模块，如window
fn:回调函数
动态加载就是你想在家那个模块，就把那个模块引进来，并不会把全部的js,css给你事先加载进来。

#提交表单
在easyui中提交表单有三种方法。
第一：传统的ajax提交
$(“#myfoem”).ajax({
type:post,
url:”userServlet”, //form提交的路径
sync:true,
catch:false,
data:$(“#foem”).serilize(), //需要提交的数据
dataType:”text”, //预期服务器的数据类型
success:function(data){
var res = $.parseJSON();//转换字符串为js脚本

$.messager({//用户操作成功提示
title:”提示信息”,
msg:”res.ok”//服务器返回的json数据

});

第二种:自动提交表单

#layout布局
这里布局的div就不写了，在api里面直接可以cv,在这里就说一些使用过程中常见的问题吧。
第一：layout是依赖于panel和resizable的，在开发中一些layout中没有的特性，都可以在他的依赖中去找。
第二：api中提供的布局是按照左西右东，上北下南的方式来指定的，我们传统的布局就是只是左边，上边和中间，那么只需要注释掉右边和下边就可以了。
但是，这里需要强调的是，center是不可以注释掉的，如果center注释掉的话，布局就会出现问题。（因为center这个并不是用region上北下南这些去定位的，是定制做的）
第三：layout中的href属性，需要注意的是 href=”xx.jsp” ， href只是吧xx.jsp中的body中的内容给你拉取过来，起的的它不管。

布局的结合使用。
场景分析：
layout :作为打的布局，也就是i整体布局，在这个布局里面除了center不能注释外，其他的根据可以根据你想要的布局去定义。
tabs和Accordion 都是作为layout中的子布局来使用的（通俗来说就是把 tabs和Accordion 放到 layout里面。）
一般使用的方法： 点击Accordion中的选项，在layout中的center(可以自己设置)中显示tabs 1…tabs n.
结合这个可以做一个小demo来测试下。
