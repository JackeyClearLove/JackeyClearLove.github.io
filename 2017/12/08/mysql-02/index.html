mysql基础

mysql运算符

1.算数运算符：+，-，*，/，%（加减乘除取余）
2.逻辑运算符：&&(and),||(或),!(非) 逻辑与（一假则假），逻辑或（一真则真），逻辑非（真为假，假为真）
3.比较运算符 : <, > , = ( <=> ), !=(<>) , between…and , IN ,not in , is null , is not null , like , regexp或rlike
< :小于 eg : 1 < 2

: 大于 ed :2 > 1
= : 等于 eg: 2=2,不能比较为null的值
<=> : 类似于等于 ，可以比较为null的值 eg : null <=> null
<> : 不等于，不能用于比较null值
between..and.. : 位于什么和什么之间， a between b and c ,eg : 2 between 1 and 3 ，返回值为ture.
in : 在什么集合之中， eg: 1 in (1,2,3)
not in :不在集合之中 1 not in (2,3,4)
is null :为空 ，
is not null : 不为空
like : 模糊匹配 ， eg : select 12345 like ‘%123%’
regexp : 正则表达式匹配 eg ； select str regexp str_path;

4.位运算符（为运算符的实质就是吧数转换为二进制，在二进制上进行逻辑操作，得出的二进制在转换为十进制）
&:位与 ，表示把一个转换为二进制的数进行逻辑与操作 2&3 转换为二进制为 10&11，逻辑与的规则是一假则假 ,1和1为1，1和0为0，所以10&11的结果为 10，转换为十进制后为2
| : 位或 ，
^ :位异或
~ ：位取反 ，eg ；常量数字1在mysql中是占8个字节，64位，那么数字1的二进制就是前面63个0后面一个1，那么去反后就是，前面63个1，后面1个0。
>> :位右移
<< :位左移

#表类型（存储引擎的选择）
常用的存储引擎： myISAM,InnoDB,MEMORY,NDB\
下面重点介绍常用的四种存储引擎
mysql5.5之前默认的引擎为myISAM
mysql5.5之后，默认的引擎为InnoDB。

MyISAM

优点 ： 速度快
缺点 ：不支持事务，外键
应用 ：若数据库频繁的select,insert为主的话，可以选择MyISAM为mysql的存储引擎。

每个MyISAM在磁盘上有三个文件，文件名与数据库名相同，单扩展名不同
XX.frm(存储表定义)
XX.MYD(MYData 存储数据)
xx.MYI(MYIndex,存储索引)

MYISAM类型的表可能会损坏，损坏的表可能不能访问

检查MyISAM表的健康状态 ： CHECK TABLE 。
修复一个损坏的MyISAM表 ： REPAIR TABLE

MyISAM类型的表支持三种不同的存储方式
1.静态表(固定长度)
2.动态表
3.压缩表
静态表是默认的存储格式，
静态表 ： 字段长度是固定的，每个字段都有固定的存储长度
优点 ：存储快，容易缓存，损坏容易修复
缺点 ：占用空间内存比较大。

静态表存储的时候是按照列的宽度来补足空格（比如定义了varchar(10),但是10里面我只用了5个，那么后面的5个就会用空格来补齐），但是在返回结果之前，空格会被去掉。前面如果有空格就是不会被去掉的。
eg : [ abcd],[abcd(后面的空格会在返回结果的时候被去掉)]。

动态表 ：字段的长度是不固定的，就是你定义的字段占多长那么就分配给你多少内存。
优点 ：节省空间。
缺点 ：出现故障修复相对比较困难 。 频繁的更新和删除容易产生碎片，需要定期的执行OPTIMIZE语句或者myisamchk-r命令来改善性能。

InnoDB
优点 ： 支持事务，提交，回滚，外键（mysql中只有InnoDB支持外键）
缺点 ： 占用磁盘空间大， 对于写的处理效率差一些。
存储方式有两种
1.共享表空间存储
.frm(表结构的存储)
innodb_data_home_dir(存放数据)
innodb_data_file_path(存放索引)
2多表空间存储
。frm(存储表结构)
。idb(分别独立的存放数据和索引)

外键（分为父表和子表）
要求父表必须有索引，子表在创建外键的时候也会自动的创建索引
注意：restrict和no action 相同，表示在子表有相关记录时，父表不能更新
cascade:表示父表在更新或删除时，更新或删除子表对应的记录
set null : 表示父表在更新或删除时，子表的记录设置为空。

当某个表被其他表创建了外键参照，那么这个表的主键不允许删除

如果你的表经常执行更新删除操作时请定期的执行optimize table tablename
查看物理盘：di -sh tablename.*;
在删除了记录之后，我们发现物理盘里面的空间还是和删除前占用的空间一样大。
当执行了optimize table tablename;后，我们菜发现删除后空间变小了
在一个，删除记录后，腾出来的空间，以后我们要向这些空间中插入数据，删除后空出来的空间会影响之后插入数据的效率。

innoDB和MyISAM索引的区别
MyISAM可以声明组合索引index(d1,d2),InnoDB不能声明。
InnoDB插入索引是插入的是 0 或者 null,索引会自动创建为自增长的下一个索引（比如现在只有一个索引为1，我现在插入了 0 或者 null , 那么给我创建的索引就是 2）
MyISAM组合索引
这个的自增长是根据先插入那个索引，那么就根据那个索引自增长
eg :
create table index_test(
d1 smallint not null,
d2 smallint notnull,
uname varchar(10) not null
);

插入记录
insert into index_test (d2,uname) values (2,”haha”),(3,”xixi”),(4,”fff”);

select * from index_test;
查询出来的记录 d1为 1，1，2
d2, : 2,3,4,
由此可以看出索引是按d2去自增长的

#如何选择合适的引擎(engine)
创建引擎
create table tablename(
name vachar(10) not null
)engine=MyISAM default charset=utf-8;

MyISAM:
如果应用以读操作和从插入数据为主而很少进行删除和更新，并且对于事务的完整性，并发行要求不高，那么合适使用MyISAM引擎。
应用 ： MyISAM在web,数据仓库，最常使用。
InnoDB :
用于十五处理，支持外键。 如果对事务的完整性有要求，在并发条件下要求数据的一致性。数据库除了查找和插入，还常常使用更新和删除，那么使用InnoDB引擎。
应用 ：经常用在计费系统，财务系统等对数据准确性要求高的系统中

#选择合适的数据类型
1.char和varchar
char属于 固定长度的字符类型
varchar属于可边长的字符类型、
char的处理速度大于varchar的速度
varchar的空间占用小于char
varchar会对尾行的空格进行处理，
char不会对尾行的空格进行处理
eg:
create table t(
v varchar(10) not null,
c char(10) not null
);

insert into t values (‘ab’,’ab’);
select concat(v,’+’),concat(c,’+’) from t;

结果 varchar : ab+ char : ab +
这样我们就可以看出varchar的速度为什么比char慢了

现如今也对varchar做了很多的速度上的处理，所以一般推荐使用varchar
对于很少变动并且要求速度的，可以考虑使用char

2.TEXT和 BLOB
text和blob都是用来存储大文本的
text ；一般就是存储文章，日记等等
blob : 允许存储二进制，一般可以存储相片，
TEXT和BLOB分别都有
text mediumtext longtext
blob mediumblob longblob

#日期类型的选择
日期类型包括 ： Data,TIME,DATATIME,TIMESTAMP
如果值存储年份，则使用一个字节的year就可以了，而不用 是同4个字节的Data.
如果存储年月日时分秒并且年份比较久远的，用DATATIME,而不是同TIMESTAMP,因为TIMESTAMP的存储的年份范围比较短。
如果存储的时间需要让不同时区的人看到，那么使用TIMESTAMP，因为日期类型中只有他能和实际时区匹配。
